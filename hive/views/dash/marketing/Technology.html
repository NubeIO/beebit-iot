How BeeBit Hive Works

The BeeBit Hive is built upon numerous industry standard technologies, allowing for simplified development, operation, and integration. The Bee is built on a Raspberry Pi, with a Pi Camera attached. Through the BeeBit developed firmware, images are sent from the camera to a person-detection algorithm running on the device.

The current person-detection algorithm that the Bees utilise is based on comparing HOGs (Histogram of Gradients), a description of what a person looks like as a set of pixel gradients in an area, to the image from the camera. However, in future we hope to implementa Machine Learning algorithm to improve the accuracy of these models, and allow more flexibility in what the Bees may be able to detect.

Once an image has been processed and a person-detection count produced, the Bee firmware sends the data back to a Nube iO Edge28 controller wirelessly, utilising LoRa technology to allow communications to occur with extremely low power usage over enormous ranges. The data from all the Bees is collected and processed by the Hive software running on the controller, and then can provide this data through an API, to the Nube iO automation software, the BeeBit frontend, or any other system where the data could be used.
